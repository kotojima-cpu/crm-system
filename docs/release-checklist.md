# リリースチェックリスト

## 概要

Staging 環境（テスト版）および本番リリース前に確認すべき項目をまとめる。

---

## テスト版（Staging）確認チェックリスト

Staging 環境にデプロイ後、以下を確認する。
URL: `https://staging--<site-name>.netlify.app`

### 認証

- [ ] ログイン画面が表示される
- [ ] admin / admin123 でログインできる
- [ ] sales01 / sales123 でログインできる
- [ ] 不正なID/パスワードでログインできない
- [ ] ログアウトできる
- [ ] 未認証状態でページアクセスするとログイン画面にリダイレクトされる

### 権限制御

- [ ] admin ユーザーでヘッダーに「ユーザー管理」リンクが表示される
- [ ] sales ユーザーでヘッダーに「ユーザー管理」リンクが表示されない
- [ ] sales ユーザーで /admin/* にアクセスすると /customers にリダイレクトされる

### 顧客一覧

- [ ] 顧客一覧が表示される
- [ ] ページネーションが動作する
- [ ] 「新規登録」ボタンが表示される（PC版）

### 顧客検索

- [ ] 顧客名で部分一致検索できる
- [ ] 住所で部分一致検索できる
- [ ] 電話番号で検索できる（ハイフンあり/なし両方）
- [ ] 検索条件をクリアできる

### 顧客詳細

- [ ] 顧客詳細画面が表示される
- [ ] 会社名、住所、電話番号、FAX、担当者情報が表示される
- [ ] 備考が表示される（存在する場合）
- [ ] 「編集」「削除」ボタンが表示される

### 顧客 CRUD

- [ ] 顧客を新規登録できる
- [ ] 顧客情報を編集できる
- [ ] 顧客を削除できる（論理削除：一覧に表示されなくなる）

### 契約一覧（顧客詳細画面内）

- [ ] 契約一覧が表示される
- [ ] 契約番号、商品名、開始日、残回数、ステータスが表示される
- [ ] 満了間近（expiring_soon）の行が黄色でハイライトされる
- [ ] 「契約を追加」ボタンが表示される

### 契約詳細

- [ ] 契約詳細画面が表示される
- [ ] プログレスバーが正しく表示される
- [ ] ステータスバッジが正しい色で表示される（有効=緑、満了間近=黄、満了=灰）

### 残回数表示

- [ ] 残回数が「残 / 全月数」形式で表示される
- [ ] 残回数が正しく計算されている（開始日 + 契約月数 + 基準日から動的算出）
- [ ] 経過月数が正しく表示される

### 契約 CRUD

- [ ] 契約を新規登録できる
- [ ] 契約情報を編集できる
- [ ] 契約を削除できる

## PWA 確認チェックリスト

### manifest

- [ ] `/manifest.json` にアクセスできる
- [ ] `name`: "OA顧客管理システム" が設定されている
- [ ] `display`: "standalone" が設定されている
- [ ] `start_url`: "/customers" が設定されている
- [ ] `icons` に 192x192 と 512x512 が設定されている

### ホーム画面追加

- [ ] Chrome DevTools > Application > Manifest でマニフェストが読み込まれている
- [ ] ブラウザのアドレスバーにインストールボタンが表示される（Chrome）
- [ ] インストール後、standalone モードで起動する

### アイコン表示

- [ ] `/icons/icon-192.png` が表示される
- [ ] `/icons/icon-512.png` が表示される
- [ ] `/icons/icon.svg` が表示される

### Service Worker

- [ ] Chrome DevTools > Application > Service Workers で sw.js が登録されている
- [ ] 静的アセット（icons, _next/static）がキャッシュされている

### PWA 版ボタン非表示

- [ ] standalone モードで「新規登録」ボタンが非表示
- [ ] standalone モードで「編集」「削除」ボタンが非表示
- [ ] standalone モードで「契約を追加」ボタンが非表示

## スマホ UI 確認チェックリスト

Chrome DevTools のデバイスモード（iPhone 14, Galaxy S21 等）で確認する。

### ヘッダー

- [ ] ハンバーガーメニューが表示される
- [ ] メニュー開閉が動作する
- [ ] メニュー内にナビリンク、ユーザー名、ログアウトボタンが表示される

### 顧客一覧

- [ ] カード UI で表示される（テーブルではない）
- [ ] 各カードにタップで顧客詳細に遷移する
- [ ] 会社名、住所、電話番号、担当者、更新日が表示される

### 顧客詳細

- [ ] 1カラムレイアウトで読みやすく表示される
- [ ] 契約一覧がカード UI で表示される
- [ ] 各契約カードに商品名、残回数、ステータスが表示される

## 本番リリース前チェックリスト

Staging での確認が完了した後、main ブランチにマージする前に確認する。

### コード品質

- [ ] `npm run lint` がエラーなしで通る
- [ ] `npm run test` が全テストパスする（30/30）
- [ ] `npm run build` がエラーなしで成功する

### 環境変数

- [ ] Netlify UI で Production 用の環境変数が設定されている
- [ ] `NEXTAUTH_SECRET` が本番用の強力な値に設定されている
- [ ] `NEXTAUTH_URL` が本番 URL に設定されている
- [ ] `DATABASE_URL` が本番 PostgreSQL に設定されている（移行後）

### セキュリティ

- [ ] .env ファイルがリポジトリにコミットされていない
- [ ] シードスクリプトのパスワードが本番用に変更されている（本番移行時）
- [ ] HTTPS でアクセスできる（Netlify は自動対応）
